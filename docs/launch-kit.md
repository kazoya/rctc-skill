## LinkedIn / X (Twitter)

```
Most people talk to AI like this:
"Write me a marketing email."
And get... generic soup.

I built a skill that fixes this.

RCTC analyzes your prompt before answering:
→ Role: who should the AI be?
→ Context: what's the situation?
→ Task: what exactly do you want?
→ Constraints: what are the limits?

It asks ONE smart question when something's missing.
Stays silent when everything's clear.
Works in Arabic + English. Cursor-ready.

Free & open source 👇
https://github.com/kazoya/rctc-skill

(Pro version coming — follow to stay updated)
```

## Reddit — r/ClaudeAI or r/PromptEngineering

**Title:**

```
I built a Claude skill that asks ONE smart question instead of answering blindly — RCTC Method [free + open source]
```

**Body:**

```
Most prompt engineering advice is "write better prompts."
RCTC flips it — Claude analyzes YOUR prompt first, then asks exactly what's missing.

Framework: Role → Context → Task → Constraints

What it does differently:
- Asks max 1 question per turn (with the reason why)
- Skips questions when context makes the answer obvious
- Learns your patterns after 5 interactions
- Works in Arabic + English
- .cursor/rules/rctc.mdc included — auto-loads in Cursor

GitHub: https://github.com/kazoya/rctc-skill

Would love feedback on where it asks too much or too little.
```
