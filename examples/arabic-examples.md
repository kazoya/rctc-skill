# أمثلة عربية — RCTC Method

## مستوى مبتدئ: برومبت ناقص جداً

### المثال 1: طلب محتوى مبهم

**المستخدم:**
```
ساعدني في كتابة شيء عن التسويق
```

**تحليل RCTC الداخلي:**
- R (الدور): ❌ غائب — من أنت؟ متخصص؟ مبتدئ؟
- C (السياق): ❌ غائب — أي نوع من التسويق؟ لمن؟
- T (المهمة): ❌ مبهم جداً — "شيء" لا يكفي
- C (القيود): ❌ غائب

**مشكلة أعلى تأثير:** المهمة مبهمة جداً

**رد RCTC:**
```
🎯 قبل أن أبدأ — سؤال مهم واحد:

لاحظت أن طلبك لا يحدد **ما الذي تريد كتابته بالضبط**.

لماذا يهمني هذا الآن؟
"شيء عن التسويق" يمكن أن يكون: مقال، بوست، خطة، سكريبت، بريد إلكتروني، 
أو عشرة أشياء أخرى — وكل واحد له أسلوب مختلف كلياً.

سؤالي:
ما نوع المحتوى الذي تحتاجه بالضبط؟ (مثال: بوست لينكدإن، مقال، بريد ترويجي)

*(قل "تجاهل" وسأختار النوع الأنسب بناءً على السياق العام)*
```

### المثال 2: طلب أفضل قليلاً

**المستخدم:**
```
اكتب لي بريد إلكتروني بارد لبيع خدماتي كمصمم جرافيك
```

**تحليل RCTC الداخلي:**
- R (الدور): ✅ مصمم جرافيك
- C (السياق): ⚠️ جزئي — نعرف المجال لكن لا نعرف العميل المستهدف
- T (المهمة): ✅ بريد إلكتروني بارد
- C (القيود): ❌ غائب

**مشكلة أعلى تأثير:** السياق — من هو العميل المستهدف؟

**رد RCTC:**
```
🎯 قبل أن أبدأ — سؤال مهم واحد:

لاحظت أن طلبك لا يحدد **من هو المستلم المستهدف**.

لماذا يهمني هذا الآن؟
البريد لصاحب متجر إلكتروني يختلف تماماً عن البريد لمدير وكالة تسويق
أو لمؤسس ستارتاب. الألم الذي تخاطبه، والمصداقية التي تبنيها، كلها تتغير.

سؤالي:
من هو العميل المثالي الذي ترسل له هذا البريد؟ (المجال، حجم الشركة، دوره)

*(قل "تجاهل" وسأكتب لـ"مدير تسويق في شركة صغيرة" كافتراض افتراضي)*
```

## مستوى متوسط: برومبت يحتاج تحسينًا

### المثال 3: برومبت جيد لكن ناقص

**المستخدم:**
```
أنت مستشار أعمال خبير. أنا أطلق منتجًا رقميًا للمعلمين في السعودية. 
اكتب لي خطة تسويقية.
```

**تحليل RCTC الداخلي:**
- R (الدور): ✅ مستشار أعمال خبير
- C (السياق): ✅ منتج رقمي، معلمون، السعودية
- T (المهمة): ✅ خطة تسويقية
- C (القيود): ❌ غائب — وهنا التباين كبير جداً

**قرار:** هل أسأل عن القيود؟

**التقييم:** خطة تسويقية بدون قيود يمكن أن تكون صفحة واحدة أو عشرين صفحة.
التباين كبير بما يكفي لأسأل.

**رد RCTC:**
```
✅ ممتاز — ثلاثة مكونات RCTC موجودة. سؤال أخير قبل البدء:

ما مستوى التفاصيل الذي تحتاجه؟
مثلاً: ملخص تنفيذي (صفحة)، أو خطة كاملة مع قنوات وميزانية وجدول زمني؟

*(قل "تجاهل" وسأكتب خطة متوسطة التفصيل)*
```

## مستوى متقدم: برومبت RCTC مكتمل

### المثال 4: برومبت احترافي كامل

**المستخدم:**
```
أنت مدير تسويق رقمي بخبرة 10 سنوات في تسويق B2B.
أنا مؤسس شركة SaaS صغيرة تستهدف شركات الخدمات اللوجستية في الخليج.
المنتج يساعد على تتبع الشحنات بالذكاء الاصطناعي. ميزانيتي محدودة.
اكتب 3 أفكار لحملات تسويقية على لينكدإن.
كل فكرة في 3 أسطر فقط. لا تستخدم مصطلحات تقنية معقدة.
```

**تحليل RCTC الداخلي:**
- R (الدور): ✅ مدير تسويق رقمي B2B خبير
- C (السياق): ✅ SaaS، لوجستيات، خليج، ميزانية محدودة
- T (المهمة): ✅ 3 أفكار حملات لينكدإن
- C (القيود): ✅ 3 أسطر، بدون مصطلحات تقنية

**رد RCTC:**
```
🚀 برومبت RCTC مكتمل — أبدأ فوراً.

---

**الفكرة 1: "يوم في حياة شاحنة"**
سلسلة محتوى تُتابع شحنة واحدة من المستودع للعميل يومياً.
تُظهر المشاكل الحقيقية التي يحلها منتجك بدون أن "تبيع" مباشرة.
الهدف: بناء الثقة قبل الطلب.

**الفكرة 2: "الرقم الذي لا أحد يتحدث عنه"**
بوستات تُبرز إحصائيات خسائر الشركات اللوجستية بسبب ضعف التتبع.
يُحدث ألم السوق أولاً، ثم تظهر كحل طبيعي.
الهدف: إيجاد الوعي بالمشكلة لدى صانع القرار.

**الفكرة 3: "شهادة بالأرقام"**
شارك نتيجة قابلة للقياس من عميل حقيقي (حتى لو صغير).
"خفضنا وقت التتبع بـ40% في أسبوعين" أقوى من أي وصف.
الهدف: إثبات اجتماعي موجه لمن يفكر في الشراء.
```

## أمثلة على التعلم والتكيف

### بعد 10 تفاعلات مع مستخدم يعمل في التسويق

**المستخدم:**
```
اكتب سكريبت لفيديو قصير
```

**RCTC (بعد التكيف):**
```
📈 بناءً على تفاعلاتنا السابقة، أعرف أنك عادةً تعمل على محتوى B2B للسوق السعودي.

سأفترض:
- الدور: مسوّق محتوى محترف
- السياق: جمهور B2B محلي
- المهمة: سكريبت فيديو قصير (60-90 ثانية)

سؤال واحد فقط: ما الهدف من هذا الفيديو؟ (توعية، تحويل، ولاء عملاء)
```
